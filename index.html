NF英文解释

First normal form (1NF)

the domain of each attribute contains only atomic values, and the value of each attribute contains only a single value from that domain.

Second normal form (2NF)

a table is in 2NF if it is in 1NF and no non-prime attribute is dependent on any proper subset of any candidate key of the table. A non-prime attribute of a table is an attribute that is not a part of any candidate key of the table.

Third normal form

the entity is in second normal form and
all the attributes in a table are determined only by the candidate keys of that table and not by any non-prime attributes.

SDLC SOFTWARE DEVELOP LIFE CYCLE

There are various software development approaches defined and designed which are used/employed during development process of software, these approaches are also referred as “Software Development Process Models” (e.g. Waterfall model, incremental model, V-model, iterative model, etc.).
There are following six phases in every Software development life cycle model:
1.Requirement gathering and analysis

  • Design
  • Implementation or coding
  • Testing
  • Deployment
  • Maintenance

Waterfall Model

Description
The waterfall Model is a linear sequential flow. In which progress is seen as flowing steadily downwards (like a waterfall) through the phases of software implementation. This means that any phase in the development process begins only if the previous phase is complete. The waterfall approach does not define the process to go back to the previous phase to handle changes in requirement. The waterfall approach is the earliest approach that was used for software development.

Good: Easy to explain to the user· Structures approach.· Stages and activities arewell defined· Helps to plan and schedule the project· Verification at each stage ensures early detection of errors / misunderstanding· Each phase has specific deliverables

Bad: Assumes that the requirements of a system can be frozen· Very difficult to go back to any stage after it finished.· Little flexibility and adjusting scope is difficult and expensive.· Costly and required more time, in addition to detailed plan

Agile Model

Extreme programming (Agile development)
Description
It is based on iterative and incremental development, where requirements and solutions evolve through collaboration between cross-functional teams.

The usage
It can be used with any type of the project, but it needs more involvement from customer and to be interactive. Also, it can be used when the customer needs to have some functional requirement ready in less than three weeks.

Good: Decrease the time required to avail some system features.· Face to face communication and continuous inputs from customer representative leaves no space for guesswork.· The end result is the high quality software in least possible time duration and satisfied customer

Bad: Decrease the time required to avail some system features.· Face to face communication and continuous inputs from customer representative leaves no space for guesswork.· The end result is the high quality software in least possible time duration and satisfied customer

What are the Structure-based (white-box) testing techniques?

Structure-based testing techniques (which are also dynamic rather than static) use the internal structure of the software to derive test cases. They are commonly called ‘white-box’ or ‘glass-box’ techniques (implying you can see into the system) since they require knowledge of how the software is implemented, that is, how it works. For example, a structural technique may be concerned with exercising loops in the software. Different test cases may be derived to exercise the loop once, twice, and many times. This may be done regardless of the functionality of the software.

What is DRE?

To measure test effectiveness a powerful metric is used to measure test effectiveness known as DRE (Defect Removal Efficiency) From this metric we would know how many bugs we have found from the set of test cases. Formula for calculating DRE is

DRE=Number of bugs while testing / number of bugs while testing + number of bugs found by user

Surrounded Regions

Given a 2D board containing ‘X’ and ‘O’, capture all regions surrounded by ‘X’.

A region is captured by flipping all ‘O’s into ‘X’s in that surrounded region.

For example,
X X X X
X O O X
X X O X
X O X X
After running your function, the board should be:

X X X X
X X X X
X X X X
X O X X

想法：
1：先走一遍，看看有没有X，要是没有X就直接退出
2：走一遍外围，碰到O就用广度优先把O变成1，
3：然后在走一遍，把剩下的O变成X
4：再走一遍，把剩下的1变成O就可以了

leedcode上会超出范围：

public class Solution {
    public void solve(char[][] board) {
        boolean hasX = false;

        for(int i = 0; i < board.length; i++){
            for(int j = 0; j < board[0].length; j++){
                if(board[i][j] == 'X'){
                    hasX = true;
                    break;
                }
            }
        }

        if(!hasX) return;

        for(int i = 0; i < board[0].length; i++){
            if(board[0][i] == 'O'){
                change(board, 0, i);
            }
        }

        for(int i = 0; i < board.length; i++){
            if(board[i][0] == 'O'){
                change(board, i, 0);
            }
        }

        for(int i = 0; i < board.length; i++){
            if(board[i][board[0].length - 1] == '0'){
                change(board, i, board[0].length - 1);
            }
        }

        for(int i = 0; i < board[0].length; i++){
            if(board[board.length - 1][i] == 'O'){
                change(board, board.length - 1, i);
            }
        }

        for(int i = 0; i < board.length; i++){
            for(int j = 0; j < board[0].length; j++){
                if(board[i][j] == 'O'){
                board[i][j] = 'X';
                }
            }
        }
        for(int i = 0; i < board.length; i++){
            for(int j = 0; j < board[0].length; j++){
                if(board[i][j] == '1'){
                    board[i][j] = 'O';
                }
            }
        }
    }

    private void change(char[][] board, int i, int j){
        if(board[i][j] == 'O'){
            board[i][j] = '1';
        }else{
            return;
        }

        if(i - 1 >= 0){
            change(board, i - 1, j);
        }
        if(i + 1 < board.length){
            change(board, i + 1, j);
        }
        if(j - 1 >= 0){
            change(board, i, j - 1);    
        }
        if(j + 1 < board[0].length){
            change(board, i, j + 1);
        }
    }

}

Generate Parentheses

Generate Parentheses
Given n pairs of parentheses, write a function to generate all combinations of well-formed parentheses.
For example, given n = 3, a solution set is:

“((()))”, “(()())”, “(())()”, “()(())”, “()()()”

想法：
1：很明显是要看各种组合的可能性，用的是递归；
2：左括号和右括号的数量相同时，一定只放左括号；
3：左括号比右括号的数量多， 并且左括号的数量小于总长度的一半时 ，可加左也可加右；
4：当左括号数量到达总长度一半时，只能用右括号
这里要注意，stringbuffer 也是immune型的数据，每次向下传入一个stringbuffer的时候都要new一个新的stringbuffer 

public class Solution {
    public List<String> generateParenthesis(int n) {

        ArrayList<String> res = new ArrayList<String>();
        StringBuffer sb = new StringBuffer();
        help(2*n, sb, 0, 0,res);
        return res;
    }

    private void help(int totLength, StringBuffer sb, int left, int right, ArrayList<String> res){
        if(sb.length() == totLength) {
            res.add(new String(sb.toString()));
            return;
        }

        if(left == right){
            sb.append('(');
            left++;
            help(totLength, sb, left,right,res);
        }else if(left > right && left < totLength/2){
            sb.append('(');
            left++;
            StringBuffer sb1 = new StringBuffer(sb);
            help(totLength, sb1, left,right,res);
            sb.deleteCharAt(sb.length() - 1);
            left--;
            sb.append(')');
            right++;
            StringBuffer sb2 = new StringBuffer(sb);
            help(totLength, sb2, left,right,res);
        }else{
            sb.append(')');
            right++;
            help(totLength, sb, left,right,res);
        }
    }    
}
