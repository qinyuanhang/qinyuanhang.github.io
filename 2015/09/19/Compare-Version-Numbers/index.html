Compare Version Numbers

Compare Version Numbers
Compare two version numbers version1 and version2.
If version1 > version2 return 1, if version1 < version2 return -1, otherwise return 0.

You may assume that the version strings are non-empty and contain only digits and the . character.
The . character does not represent a decimal point and is used to separate number sequences.
For instance, 2.5 is not “two and a half” or “half way to version three”, it is the fifth second-level revision of the second first-level revision.

Here is an example of version numbers ordering:

0.1 < 1.1 < 1.2 < 13.37

想法：
把字符串按照“.”拆开，重头到尾比较数字的大小，只要碰到大的那个就返回对应的结果
注意可能有“1”和“1.0.0.0.1”这样的情况，所以按照长度短的那个来比较，
要是有“1”和“1.0.0.0”这样的贱题目，就要比较长度长的后面的所有数字，要是碰到比0大的，就直接返回对应的结果

*注意

###java中split小数点的时候，要这样split（”\.”）要转义一下。要不然翻个出来的是空的数组

public class Solution {
    public int compareVersion(String version1, String version2) {
        String[] v1 = version1.split("\\.");
        String[] v2 = version2.split("\\.");

        int l = v1.length >= v2.length ? v2.length : v1.length;

        for(int i = 0; i < l; i++){
            if(Integer.parseInt(v1[i]) > Integer.parseInt(v2[i])){
                return 1;
            }else if(Integer.parseInt(v1[i]) < Integer.parseInt(v2[i])){
                return -1;
            }
        }
        int possible = v1.length > v2.length ? 1 : -1;
        String[] longer = v1.length > v2.length ? v1 : v2;
        for(int i = l; i < longer.length; i++){
            if(Integer.parseInt(longer[i]) > 0) return possible;
        }
            return 0;
    }
}
